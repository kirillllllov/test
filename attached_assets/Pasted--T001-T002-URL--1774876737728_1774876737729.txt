Уточнение к плану: T001 и T002
Принцип работы с URL коннекторов
Каждый коннектор — это отдельный сервис с собственным хостом. Пути у всех коннекторов одинаковые, различие только в базовом URL.

Исходящие вызовы (Модуль → Коннектор):

VK_CONNECTOR_URL=http://localhost:5001
TELEGRAM_CONNECTOR_URL=http://localhost:5002
Отправить сообщение в VK:
  POST http://localhost:5001/message
Создать клавиатуру в Telegram:
  POST http://localhost:5002/keyboard/create
Обновить клавиатуру в VK:
  POST http://localhost:5001/keyboard/update
Входящие вызовы (Коннектор → Модуль):

Поскольку все коннекторы шлют на одни и те же пути модуля (/command, /user_message и т.д.), модуль не может различить их по URL. Поэтому каждый коннектор передаёт своё имя через заголовок:

POST http://localhost:5000/user_message
Header: X-Connector-Name: vk
Это не меняет API-контракт (тела запросов остаются по спецификации), но даёт модулю однозначную идентификацию источника.

T001 — обновлённый
src/connectors/http-connector.ts:

name — строка ("vk", "telegram")
baseUrl — http://localhost:5001
sendMessage(...) → POST http://localhost:5001/message
createKeyboard(...) → POST http://localhost:5001/keyboard/create
updateKeyboard(...) → POST http://localhost:5001/keyboard/update
Никаких имён коннекторов в путях — только чистый baseUrl + стандартный путь.

T002 — обновлённый
Входящие маршруты модуля остаются без изменений:

POST /keyboard/input
POST /image
POST /command
POST /user_message
POST /message/action
Идентификация коннектора — через заголовок X-Connector-Name, который каждый коннектор обязан передавать. Модуль читает его в начале обработки и сохраняет в контекст машины состояний как connectorName. Если заголовок отсутствует — возвращает 400 Bad Request.

Middleware для извлечения имени коннектора:

src/middleware/connector-name.ts
Читает X-Connector-Name из заголовка → кладёт в req.connectorName → все обработчики уже знают, кому отвечать.

Итоговая схема
VK Connector (http://localhost:5001)
  ← GET /command  (Header: X-Connector-Name: vk)
  Module обрабатывает, знает что это vk
  → POST http://localhost:5001/message  ← ответ уходит на хост VK
Telegram Connector (http://localhost:5002)
  ← POST /user_message  (Header: X-Connector-Name: telegram)
  Module обрабатывает, знает что это telegram
  → POST http://localhost:5002/keyboard/create
